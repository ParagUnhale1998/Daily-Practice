it change the default format of date to main formet just using date in this code

UpperCasePipe: This pipe transforms a string to uppercase. It takes a string as input and returns the uppercase version of that string.

'hello world' | uppercase => {{ "hello world" | uppercase }}

LowerCasePipe: Similar to UpperCasePipe, this pipe transforms a string to lowercase. It takes a string as input and returns the lowercase version of that string.

'Hello World' | lowercase => {{ "Hello World" | lowercase }}

CurrencyPipe: The CurrencyPipe formats a number as a currency using currency symbols, separators, and specified formatting options. It is commonly used for displaying monetary values.

{{ cost | currency : "INR" : "symbol" : "1.2-2" : "en-IN" }}

code => cost | currency:'INR':'symbol':'1.2-2':'en-IN'

'INR' specifies the currency code for Indian Rupee.

'symbol' indicates that the currency symbol should be displayed.

'1.2-2' specifies the minimum and maximum fractional digits to be displayed.

'en-IN' sets the locale to English India for proper currency formatting.

DecimalPipe: This pipe is used for formatting numbers as decimals. It allows you to specify the decimal separators, the number of decimal places, and other options for formatting.

{{ temperature | number : "1.1-1" }}°C

'1.1-1' specifies the minimum and maximum fractional digits to be displayed.

The first digit before the dot (1 in this case) represents the minimum number of integer digits that should be displayed. If the number has fewer digits, leading zeros will be added.

The digit after the dot (1 in this case) represents the minimum number of fractional digits to be displayed. If the number has fewer digits, trailing zeros will be added.

The last digit after the hyphen (-1 in this case) represents the maximum number of fractional digits that can be displayed. If the number has more digits, it will be rounded or truncated to fit within the specified range.

°C is appended to the formatted temperature to indicate the unit (Celsius).

JsonPipe: The JsonPipe is used to convert a value into a JSON string representation. It is commonly used for debugging or displaying JSON data. 

                    {{ jsonPipe | json }}

{{ { name: "John", age: 30, city: "New York" } | json }}

PercentPipe: The PercentPipe is used to format a number as a percentage. It multiplies the input value by 100 and adds the percentage symbol.

{{ 0.75 | percent }}

SlicePipe: The SlicePipe is used to extract a portion of an array or string. It takes start and end indices as arguments and returns a new array or string with the extracted portion.

{{ "Hello, World!" | slice : 0 : 5 }}

AsyncPipe: This pipe is used for handling asynchronous data. It subscribes to an observable or a promise and automatically updates the view with the latest value emitted by the observable or promise.

{{ data$ | async }}

KeyValuePipe: This pipe transforms an object or map into an array of key-value pairs. It is useful for iterating over object properties in the template.

Key: {{ item.key }}, Value: {{ item.value }}

TitleCasePipe: The TitleCasePipe converts the first character of each word in a string to uppercase and the rest of the characters to lowercase. It is often used for displaying properly formatted titles or headings.

{{ "hello, world!" | titlecase }}

I18nPluralPipe: This pipe is used for handling pluralization of messages based on numeric values and locale rules. It selects the appropriate plural form of a message based on the provided value.

{{ numItems }} {{ numItems | i18nPlural : { "=0": "No items", "=1": "One item", other: "# items" } }}

singletons

{{ In Angular, a singleton is a design pattern used to create a single instance of a class that can be shared and accessed throughout the application. It ensures that only one instance of the class is created and that all subsequent requests for an instance of that class return the same instance. In Angular, singletons are typically implemented using Angular's dependency injection (DI) system. When a service is registered with the DI system at the root level or using the @Injectable({ providedIn: 'root' }) decorator, Angular creates a single instance of that service and injects it wherever it is requested. Benefits of using singletons in Angular: Shared state: Singleton instances allow data and state to be shared across different components, services, and modules in the application. Efficiency: Creating a single instance of a service can improve performance and memory usage by reusing the same instance instead of creating multiple instances. Consistency: By using singletons, you ensure that the behavior and state of the service remain consistent throughout the application. Centralized management: Singletons provide a centralized way to manage and control shared resources and functionality. It's important to note that Angular's DI system handles the creation and management of singletons automatically when services are registered at the root level. However, if you register a service at a component level or with a specific module, a new instance will be created for each component or module. Overall, singletons in Angular are a powerful tool for creating shared and reusable instances of services, allowing for efficient communication and consistent state management within an application. }}

The optional chaining operator (?.) is a feature introduced in JavaScript to simplify accessing nested properties or calling methods on objects when the intermediate properties may be null or undefined. It helps prevent "TypeError: Cannot read property 'x' of null" errors. Here's a simple example to illustrate its usage: javascript Copy code const person = { name: 'John', age: 25, address: { city: 'New York', zipCode: 12345 } }; console.log(person.address?.city); // Output: 'New York' console.log(person.address?.street); // Output: undefined console.log(person.phoneNumber?.toString()); // Output: undefined In the above example, person is an object with properties name, age, and address. The address property is another object with properties city and zipCode. By using the optional chaining operator (?.), we can safely access nested properties or call methods. If any intermediate property along the chain is null or undefined, the expression will short-circuit and return undefined instead of throwing an error. In the example, person.address?.city successfully accesses the city property because address is defined. However, person.address?.street returns undefined since street doesn't exist. Similarly, person.phoneNumber?.toString() returns undefined because phoneNumber is not defined in the person object. This allows for safer and more concise code when dealing with potentially nullable properties or methods in JavaScript. }}
